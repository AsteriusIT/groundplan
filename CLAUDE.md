# CLAUDE.md

Guidance for Claude Code (and humans) working in this repository.

## What this is

**groundplan** (repo/CLI/extension working name; public name pending — see
"Naming") turns Terraform and Kubernetes into living, interactive, reviewable
architecture diagrams. Teams review infrastructure changes visually (with blast
radius, exposure and permission risks), keep documentation permanently in sync
with code, and see their cloud through the lens that matches their question —
network, permissions, C4 systems — instead of thousands of lines of HCL.

**Trust model (the moat): we ingest data, not access.** The platform reads the
plan JSON and CI-rendered manifests the user's own pipeline produces. It never
holds cloud credentials, never reads Terraform state, never runs `terraform`,
`helm` or `kustomize`. The one deliberate extension: an optional **read-only**
kubeconfig for live-cluster views (encrypted like PATs, LIST-only, Secret
values never fetched). **Never introduce cloud SDK credentials or state access.**

### Product surface (all shipped)

- **Visual PR review**: plan.json → graph, implicit deps, impact propagation,
  attribute before/after diff (masked), deterministic change summary,
  `Exposed`/`Privileged` risk badges, idempotent PR comments on
  GitHub/GitLab/Azure DevOps with embedded changes-only PNG.
- **Living docs of main**: static HCL parse, auto-regenerated by a ref poller
  on merge; history timeline + compare-two-versions; PR lifecycle (soft-close
  on branch deletion).
- **Five lenses on one graph** (`?view=`): `infra` (raw), `adapted`, `c4`,
  `network` (vnet⊃subnet⊃resource + NSG/exposure), `iam`
  (principal→role→scope). Deep semantics are **Azure-first**; any provider
  gets nodes/deps/modules/diffing.
- **Annotation layer**: note/link/group/hide/rename beside (never inside) the
  snapshot; adapted & C4 projections; orphan reconciliation; AI proposals in a
  review inbox.
- **AI layer (opt-in)**: PR summaries, docs "Explain", annotation proposals,
  guided tours; plus the experimental Azure-only **AI Infrastructure Studio**
  (chat → HCL → live diagram + lint + zip). `AI_API_KEY` is the flag.
- **Kubernetes**: manifests repos (`iac_type`) reviewed like Terraform via
  graph-vs-graph diff; live clusters (top-level, read-only) drawn per namespace.
- **Exports & sharing**: server-rendered SVG/PNG (cached), editable draw.io
  export (with a generated shape library), Confluence publish (idempotent page
  per repo), public revocable share links.
- **Tenancy**: organizations + RBAC (owner/admin/member), invitations,
  `SINGLE_ORG` self-host mode, OIDC resource-server auth, branded Keycloak.
- **Developer experience**: `@asteriusit/cli` (`groundplan push-plan`, published
  on npm), VS Code extension with live preview + git-baseline diff mode,
  multi-file HCL/YAML playground with drafts.
- **Delivery**: prod docker-compose (Caddy auto-TLS), Helm chart + kind smoke
  CI, GHCR/Scaleway images, Trivy gate, Astro marketing website (noindex until
  naming clears).

## Commands

Run from the repo root. Package manager is **pnpm** (v10). Node **>= 22**
(`.nvmrc` pins 24). Local dev needs Postgres: `docker compose up -d`
(add `--profile auth` for Keycloak on :8085).

| Command | What it does |
| --- | --- |
| `pnpm install` / `pnpm dev` / `pnpm build` / `pnpm typecheck` / `pnpm test` / `pnpm clean` | Workspace-wide |
| `pnpm dev:backend` / `pnpm dev:frontend` | One app (ports 3000 / 5173) |
| `pnpm keycloak:build` | Build the Keycloak theme provider jar |
| `pnpm --filter @groundplan/backend test\|migrate\|db:generate` | Backend tests / migrations |
| `pnpm --filter @groundplan/backend drawio:icons\|drawio:library` | Regenerate draw.io assets (`:check` variants gate CI) |
| `pnpm --filter groundplan-vscode package` | Build the `.vsix` |
| `deploy/chart/tests/run.sh --update` | Refresh Helm golden files after any chart edit |

- **Backend route tests must run via `pnpm test`** (it sets `NODE_ENV=test`);
  a bare `node --test` picks up `.env` OIDC and 401s every inject.
- Frontend tests: vitest + Testing Library (jsdom) + `vitest-axe`; mock `fetch`.
- Follow TDD; tests live beside their subject as `*.test.ts`.

### Ports & dev wiring

- Backend on **:3000**. Business routes live under **`/api/v1`**; org-owned
  resources are nested under **`/api/v1/orgs/:orgId/...`** (see Tenancy).
  Probes at the root: `GET /healthz` (liveness, no DB) and `GET /readyz`
  (503 when Postgres is unreachable).
- Frontend dev on **:5173**, proxying `/api` → `http://localhost:3000` — in-app
  always call `/api/...`, never hardcode the backend origin.

## Conventions

**General**
- TypeScript `strict` everywhere, plus `noUncheckedIndexedAccess` and
  `noUnusedLocals/Parameters`. Don't loosen these — fix the code.
- Config extends `tsconfig.base.json`; shared options go there.
- One commit per Jira story: `feat(scope): summary (GP-xx)`; direct-to-main.

**Backend**
- ESM + `NodeNext` ⇒ **relative imports use `.js` extensions** even from `.ts`
  files. Required, not a mistake.
- `buildApp` stays separate from `listen()`; every collaborator is injectable
  through `buildApp(env, opts)` — pool, `verifyConnection`, `ai`, poller clock,
  k8s reader, confluence client — so tests run offline. Never write a test
  that calls a real model or network.
- All env access goes through `config/env.ts`. Fail-closed in production:
  boot refuses without OIDC config and `ENCRYPTION_KEY`; dev/test get safe
  defaults.
- Drizzle over node-postgres; the `db` plugin owns the pool and decorates
  `app.pool`/`app.db`. Migrations are **generated** (`db:generate`), never
  hand-written, and run under a Postgres advisory lock.

**Secrets (uniform rules — PATs, kubeconfigs, Confluence credentials)**
- Encrypted at rest (AES-256-GCM, `lib/encryption.ts`), **write-only**: masked
  as `"***"` through the `toPublic*` mappers — always map rows through them.
- `webhook_token` is shown once at create, then excluded; per-repo and
  app-wide tokens are rotatable. Compare tokens with `safeEqual`
  (constant-time); invite tokens are stored as SHA-256 hashes.
- Never log an authenticated clone URL (errors are token-redacted).

**Frontend**
- Imports via `@/`; styling is Tailwind v4 — all theme tokens live in
  `src/index.css`. **Never hardcode a colour**; use the semantic utilities
  (`bg-create-soft`, `text-impacted`, …) — `design-tokens.test.ts` enforces it.
  Three themes: light, blueprint, **carbon (default)** via `.dark` +
  `data-theme`. Fonts: `font-display` / sans / `font-mono` (data).
- All HTTP goes through `src/api/client.ts` (org-scoped: it prefixes
  `/orgs/:orgId` from the active org) — never `fetch` directly in components.
  Types in `src/api/types.ts` mirror backend responses field-for-field.
- Auth: OIDC Authorization Code + PKCE (`src/auth/`), auto-redirect to the IdP
  (no `/login` page). New protected pages just render inside the guarded
  layout — don't re-check auth per page.
- App shell: `AppLayout` sidebar + flat `<Outlet>`; nav is Dashboard /
  Projects / **Clusters** / Playground / Settings. `.blueprint-grid` paper is
  opt-in per diagram view. Theme picker lives in Settings **only**.
- Renderer principles: diff-only colour (structure stays neutral), per-view
  camera (`planCamera`), view pills ≠ tool toggles, legend owns bottom-left,
  viewport preserved on refresh, ghost-the-unchanged in diff mode.
- shadcn/ui components via `pnpm dlx shadcn@latest add <name>` (in
  `apps/frontend`); AI prose renders through react-markdown (`AiResponse`),
  never `rehype-raw` — model output is untrusted input, never HTML.
- Dev-only `/styleguide` route renders every token + primitive.

## Architecture: things to know before extending

**Graph engine.** One shared format, many producers. `GraphSnapshot` types +
validator live in `packages/graph-parser` — schema **v1..v8, versioned in
place** via optional fields (v5 annotations, v6 IAM, v7 flattened
`attributes`, v8 verbatim HCL `source`). Bump the version only when a new
optional field is populated so old snapshots stay byte-identical. Producers:
**A** plan.json (`graph/plan-parser.ts`), **B** static HCL
(`@groundplan/graph-parser`, entrypoint moved by `terraform_path` like
`-chdir`), **C** Kubernetes objects (`graph/k8s-mapper.ts` — one mapper for
live reads, repo YAML and CI-rendered output; node ids are
namespace-qualified). Consumers must handle any version. Backend
`src/graph/` also holds: impact propagation, `diff.ts` (docs compare) vs
`change-diff.ts` (K8s head-vs-main colouring — different questions, don't
merge them), attribute diff, network/IAM extraction, adapted/C4 projections,
ELK layout + SVG/PNG render, draw.io converter, `summarize.ts`
(deterministic Markdown), `hcl-lint.ts` (studio rules). Prefer deterministic
rendering everywhere: same input ⇒ same diagram, byte-for-byte.

**Tenancy & RBAC (GP-112..118).** `plugins/org-scope.ts` registers all
org-owned routes under `/api/v1/orgs/:orgId` and enforces membership —
cross-tenant requests get **404, never 403** (no existence leaks). Roles
owner > admin > member; the permission matrix lives in backend
`src/rbac/permissions.ts` and is mirrored in frontend `src/rbac/` (`useCan`).
Never remove the last owner. `SINGLE_ORG=true` = self-host mode (first user
owns, others auto-join). Invitations are copy-link (no SMTP), role-scoped,
expiring. Beware the Fastify parametric-prefix gotcha: a child plugin
registered with its own `:param` prefix drops the parent's params — the
org-scope plugin handles this; follow its pattern for new nested routes.

**GitOps loop (GP-106..111).** The ref poller (`services/ref-poller.ts`,
plugin-scheduled) tracks remote branches, auto-regenerates docs of main on
merge, and soft-closes PRs whose branch vanished (history kept). CI pushes
plans via `POST /api/v1/webhooks/ci/:repositoryId` (`X-Groundplan-Token`,
10 MB cap) — the CLI wraps this with provider auto-detection. Webhooks stay
auth-exempt; everything else is covered by the global auth hook
(`plugins/auth.ts`, JIT user upsert).

**Annotations (GP-55, GP-70..77).** Five types in one table, anchored to
Terraform addresses, `status` resolved/orphaned/proposed, `provenance`
human/ai (permanent). Reconciliation flips status, never deletes, and
reverses itself; proposals move to resolved only via explicit PATCH. The
adapted projection (`graph/adapted.ts`) is a pure fold returning an ordinary
GraphSnapshot; C4 = `?granularity=group`. Annotate on the raw view only;
proposals render in the `ProposalInbox`, never on canvas.

**AI layer (GP-61..65, 75, 78/79-tours, 136..143).** `AI_API_KEY` is the
feature flag (unset ⇒ status reports disabled, routes 404, zero frontend
surface; `AI_MODEL` defaults to `claude-opus-4-8`). **The model never sees
raw plan JSON or repo HCL** — only deterministic Markdown briefs from
`services/ai-input.ts` (golden-tested). Prompts are versioned files in
`apps/backend/prompts/` — content hash = cache version. Cache key
`(kind, target, prompt version, model)`; failures never cached; one
generation in flight per target (409). Generation is always user-triggered,
streamed, labelled with the model name, and excluded from share links.
Guided tours (`services/tour.ts`, `routes/tours.ts`) return camera scripts
validated against the snapshot. The **AI Studio** (`routes/ai-studio.ts`,
`/studio` page) is ephemeral (nothing stored), Azure-focused, flagged
Experimental, and lint-checked deterministically (GP-139).

**Kubernetes.** Live clusters are **top-level org resources** (no project;
sidebar: Clusters) — a manifests *repository* stays inside its project
because PR review is what projects are for. Repos declare
`iac_type: terraform | kubernetes` (immutable, set at attach);
`terraform_path` doubles as the manifests directory. Snapshot sources mirror
the Terraform pair: `k8s_manifest` (docs of main) / `k8s_rendered` (PR head,
CI-rendered) / `k8s_namespace` (live) — `services/graph-snapshots` owns the
mapping (`DOCS_SOURCES`/`PR_SOURCES`); never re-derive it in a `where`. No
plan ⇒ diff-by-comparison against the latest docs of main
(`changesFromBase`, recorded in `stats.base`). We never run helm/kustomize;
a body we cannot read is a 422 that stores nothing. Secret *values* never
reach a graph. K8s snapshots get the diagram and the deterministic summary
only — `viewsFor()` is the one place that decides; no annotate/AI/tours/share.

**Exports & integrations.** SVG/PNG server-rendered and cached (GP-37);
draw.io export (GP-173..177) embeds vendored icons and mirrors canvas edge
semantics — regenerate assets with the `drawio:*` scripts (CI checks them);
shape library must load from Device, not URL. Confluence (GP-178, complete):
the credential lives on an **org-level Integration** (`integrations`,
GP-183 — encrypted, write-only, `integration:manage` = admin, verify checks
credential + base URL), attachable by N repos; a `confluence_connections` row
is now a pure per-repo target (`integrationId` + `spaceKey`). Publish renders
a 2× PNG (GP-182) and a summary as one idempotently-updated page with a "View
live in Groundplan" backlink (a `docs_latest` share link when one exists, else
the in-app docs URL); failures land in `last_publish_error`, never thrown. PR
comments have one adapter per provider behind `pr-comment-port.ts`.

**VS Code extension (GP-144, GP-151).** Live preview: parse in the extension
host with `@groundplan/graph-parser`, draw with `@groundplan/canvas` in a
strict-CSP Vite-built webview; debounced re-parse of dirty buffers,
last-good graph + out-of-sync chip on errors, Problems-panel diagnostics,
node↔code navigation off `node.source` ranges. Diff mode: git baseline
(HEAD or merge-base, per-SHA cache) through `@groundplan/graph-differ`,
ghost-the-unchanged, changed-only filter. Settings (`groundplan.theme`,
`rootDir`, `followActiveFile`) are preferences, not chrome. **Fully offline
— never add a network call or telemetry.** Publishing:
`docs/vscode-publishing.md` (tag `vscode-v<version>`).

**Website (GP-157..166).** `apps/website` is customer-language only — no env
vars, 404s or GP numbers; tests pin copy verbatim against
`docs/website-presentation.md` and the built dist. Single-theme carbon.
Assets are real product output (`assets-src/render-hero.ts`). Noindex until
naming clears.

**Deployment.** Self-host = `docker-compose.prod.yml` (Caddy auto-TLS +
frontend + api + migrate + Postgres + branded Keycloak) or the Helm chart
(`deploy/chart/groundplan`): external DB/IdP by default, eval-only embedded
alternatives that `fail` rendering when combined with their external
counterpart; migrations as a hook Job. After any chart edit run
`deploy/chart/tests/run.sh --update`; CI lints, goldens and kind-smokes.
